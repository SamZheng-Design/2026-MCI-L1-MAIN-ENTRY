## 核心业务逻辑（Y型流程）

```
                    ┌──────────┐
                    │  身份通   │  ← 统一入口
                    │ Identity │
                    └────┬─────┘
                         │
                    ┌────┴────┐
                    │  分 流   │
                    └─┬─────┬─┘
            ┌───────┘       └───────┐
            ▼                       ▼
    ┌──────────────┐      ┌──────────────────┐
    │   融资者路径   │      │    投资者路径      │
    │   申请通      │      │  评估通(筛子)      │
    │ Application  │      │  风控通(筛子)      │
    └──────┬───────┘      │  机会通(看板)      │
           │              └─────────┬────────┘
           └────────────────────────┤
                                    ▼
                          ┌──────────────┐
                          │  条款通→合约通  │  ← Y型汇合
                          │  结算通→履约通  │
                          └──────────────┘
```

### 关键设计要点
1. **身份通是统一入口**：所有用户通过身份通认证后，按角色分流
2. **融资者只需上传数据**：通过申请通整理/上传信息，数据进入投资者筛选池
3. **评估通+风控通是投资者的AI筛子**：每个投资者可自定义投资标准和核验方式
4. **机会通是投资者统一看板**：展示筛后项目，不设筛子=看到全部项目
5. **条款通是Y型汇合点**：投融资双方开始协同
